## Critical Fixes Required

### 1. Flight Tracking - Full Global Coverage

```
PROBLEM: Only 2 flights displaying is completely wrong.

REQUIREMENTS:

Data Source Implementation:
- Use OpenSky Network API: https://opensky-network.org/api/states/all
  - Returns ALL aircraft currently airborne (typically 5,000-15,000+)
  - Free tier: 400 credits/day, anonymous: 100 requests/day
  - Each request returns global state vectors

- Fallback: ADS-B Exchange API
  - https://www.adsbexchange.com/data/
  - Military aircraft focus
  - Requires API key for full access

Implementation Fix:
1. Fetch ALL aircraft from OpenSky, not just a sample
2. Filter client-side for performance, not server-side truncation
3. Implement clustering at low zoom levels (show counts, not individual planes)
4. At high zoom, show individual aircraft with proper icons

Aircraft Volume Expectations:
- Global view: 8,000-15,000 aircraft visible
- Regional view (Europe): 2,000-4,000 aircraft
- Conflict zone focus: 50-500 aircraft depending on area
- Military filter: 100-500 trackable military aircraft globally

If you're only seeing 2 flights, check:
- API response is being parsed correctly
- You're not limiting results in the query
- Pagination is handled if API paginates
- Filter isn't accidentally too restrictive
```
